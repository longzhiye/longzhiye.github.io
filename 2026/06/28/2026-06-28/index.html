Android 16 SystemUI：锁屏界面锁图标大小调整

背景

在 Android 锁屏（Keyguard）界面中，底部中央通常会展示一把锁图标，用于提示当前处于锁定状态，并配合生物识别 / 滑动解锁等交互。

该图标的实际显示大小并不完全由 drawable 本身决定，还会受到 SystemUI 中 lock_icon_padding 维度资源的影响。padding 越大，图标在可用区域内被“内缩”得越多，视觉上就越小。

在 Android 16 定制过程中，我们发现默认的 48px padding 会让锁图标偏小，与锁屏整体视觉比例不够协调。因此通过修改 SystemUI 资源，将其调整为更合适的尺寸。

修改后

改动说明

本次修改只涉及 SystemUI 的一处维度资源，改动面非常小，适合作为显示效果类定制的轻量方案。

原理简析

在 Keyguard 锁图标布局中，lock_icon_padding 会作为图标四周的内边距参与测量与绘制：

  • padding 较大（48px）：图标内容区被压缩，视觉尺寸偏小
  • padding 较小（12px）：图标内容区增大，视觉尺寸更明显

因此，这不是替换锁图标资源，也不是改布局宽高硬编码，而是通过调整 padding 间接控制图标最终呈现大小。这种方式的好处是：

  1. 改动集中：只需改资源文件，不动 Java/Kotlin 逻辑
  2. 风险较低：不涉及解锁流程、生物识别等业务代码
  3. 便于回滚：改一个 dimen 即可恢复默认效果

适用场景

适合以下定制需求：

  • 锁屏底部锁图标过小 / 过大，影响品牌视觉一致性
  • 希望在不替换 drawable 的前提下快速调图标观感
  • 需要在 Android 16 SystemUI 上做轻量 UI 微调

如果后续还要适配不同分辨率、异形屏或折叠屏，建议结合实际机型再评估是否需要补充 values-swXXXdp 等资源覆盖。

小结

这次 SystemUI 定制的核心非常简单：把 Keyguard 锁图标的 lock_icon_padding 从 48px 调整为 12px，从而放大锁屏底部锁图标的视觉尺寸。

对于 Android 16 这类显示效果类需求，优先从 dimens.xml 这类资源层入手，往往比深入改布局或业务逻辑更高效、也更安全。
